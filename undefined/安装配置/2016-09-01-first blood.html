First Blood

发表于 2016-09-01   |   分类于 安装配置   |     |   阅读次数

机缘巧合，前几天学习了应用CocoaPods建立私有仓库并向王大胖子吹嘘之，然后他问他怎么样才能看到呢，于是此博客应运而生。应用Hexo搭建博客之后，选择了视觉最为简单的Next主题，世界瞬间干净了许多。
首先记录一下搭建的过程吧，虽然Hexo已经有了详细的文档。

背景环境

Node.js

Git

Github账户

安装Hexo

cd一个文件夹，作为博客根目录

安装：

sudo npm install -g hexo

初始化：

hexo init

生成静态页面：

hexo generate

启动本地服务：

hexo server

然后浏览器输入http://localhost:4000就可以本地调试了!

配置Github

默认已经有了账户，建立一个仓库，名称必须为【your_user_name.github.io】

进入博客根目录找到_config.yml文件，翻到最下面，改成类似这样：

deploy:
 type: git
 repository: https://github.com/kaisayoung/kaisayoung.github.io
 branch: master

关联：

npm install hexo-deployer-git –save

部署：

hexo deploy

此时就可以打开你的博客了！

https://kaisayoung.github.io/

其他一些诸如换头像，改名字，选语言，增加评论，站内搜索，统计阅读量等功能Next主题都有详细的使用说明，忍不住想给点个赞。

总结

最常用的几个命令

创建新文章：

hexo new “New Post”

启动本地服务：

hexo server

此时可以本地先体验测试一下

生成静态页面：

hexo generate

部署到远端：

hexo deploy

生成静态页面前还可以清除一下缓存：

hexo clean

这样就算是正式生效了
